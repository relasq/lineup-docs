## Autenticação

Todas as rotas <code>/v1</code> usam token de API da Lineup.

```http
Authorization: Bearer lup_seu_token
```

<Warning>
  O token <code>lup_</code> representa a conta dona do workspace. Ele não é o
  token do cliente final, não é o Bot Token do Discord e não deve ficar no
  navegador.
</Warning>

## Separação das rotas

<Tabs>
  <Tab title="Core API">
    Rotas para Inboxes, tickets sem Discord, mensagens, transcripts e
    solicitações de ferramentas da IA.

    ```http
    /v1/inboxes
    /v1/inboxes/:inboxId/tickets
    /v1/inboxes/:inboxId/tickets/:ticketId/messages
    ```

  </Tab>
  <Tab title="Discord Plugin">
    Rotas para automação de tickets vinculados ao bot Discord. O nome do path
    continua <code>/v1/bots</code> por compatibilidade, mas o produto trata isso
    como plugin da Inbox.

    ```http
    /v1/bots
    /v1/bots/:botId/tickets
    /v1/bots/:botId/tickets/:ticketId/participants
    ```

  </Tab>
  <Tab title="Chat público">
    Rotas usadas pelo frontend do chat. Elas usam cookies e tokens temporários,
    não token <code>lup_</code>.

    ```http
    /chat/tickets
    /chat/:ticketId/session
    /chat/:ticketId/anonymous
    /chat/:ticketId/access
    ```

  </Tab>
</Tabs>

## Respostas e erros

<AccordionGroup>
  <Accordion title="200">
    Operação concluída. A resposta varia por endpoint.
  </Accordion>
  <Accordion title="400">
    Payload inválido, operação não permitida ou estado incompatível.
  </Accordion>
  <Accordion title="401">
    Token ausente, inválido, revogado ou sessão pública expirada.
  </Accordion>
  <Accordion title="403">
    A conta ou pessoa autenticada não tem permissão para a ação.
  </Accordion>
  <Accordion title="404">
    Inbox, plugin, ticket, mensagem ou solicitação não encontrada.
  </Accordion>
</AccordionGroup>

## Exemplos rápidos

<CodeGroup>

```bash Listar Inboxes
curl "https://api.lnup.pro/v1/inboxes" \
  -H "Authorization: Bearer lup_seu_token"
```

```bash Criar ticket
curl -X POST "https://api.lnup.pro/v1/inboxes/inbox_123/tickets" \
  -H "Authorization: Bearer lup_seu_token" \
  -H "Content-Type: application/json" \
  -d '{"customerName":"Maria","subject":"Ajuda","reason":"Preciso de suporte."}'
```

```bash Enviar mensagem
curl -X POST "https://api.lnup.pro/v1/inboxes/inbox_123/tickets/ticket_123/messages" \
  -H "Authorization: Bearer lup_seu_token" \
  -H "Content-Type: application/json" \
  -d '{"authorRole":"agent","authorName":"Equipe","content":"Vamos verificar."}'
```

</CodeGroup>
